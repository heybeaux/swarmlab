Three views ship: a lab index grid that reads real `experiments/*` folders, an experiment detail page rendering the README via `marked` with per-run stats, and the hero trace replay — agents as glowing nodes on an SVG stage, messages flying the bus with topic labels, score events pulsing amber rings from center into a telemetry panel, kills graying nodes with a rose ring, all driven by a rAF loop with play/pause/speed/scrubber and a clickable event log. The surprise: real traces are unwatchable raw — the smoke run spans 3ms, so a naive timestamp-scaled replay collapses into a single frame. The fix was a virtual timeline (`src/lib/replay.ts`) that preserves order and rough spacing but enforces a minimum visual gap per event; this will matter even more for LLM-backed experiments where bursts of near-simultaneous bus traffic are the norm. Verified live in a browser mid-replay (screenshot: `docs/observatory-replay.png`): dark observatory vibe landed, build + strict typecheck green, zero `any` in public signatures. ## 02 — Adversarial Pair
Built `experiments/02-adversarial-pair/` — a coder and a breaker (test-writer) alternate on the same `classify(n)` file: the breaker probes the live code for an input where it disagrees with a hidden FizzBuzz-family oracle and files one failing test; the coder rewrites to pass the whole suite. The honesty seam is that the coder's artifact is an ordered rule list that both *compiles to a real executable function* (the harness runs every test for real) and *renders to readable TypeScript* (stored verbatim in the trace) — so `pass` is always executed, never claimed. Two surprises. First, the fair fight (poison=0) *converges and disarms itself*: the breaker exhausts every honest bug in 3 rounds, gives up, churn→0, code===oracle — adversarial pressure hardening code to a fixed point, no explosion. Second and sharper: a `PAIR_POISON` knob lets the breaker lie about the expected label with some probability, and at 0.3 the run stays **green the entire time (passRate=1.0) while shipping a demonstrably wrong program** (`oracleConsistent=0` at 7 inputs) — because each lie *vetoes* the coder installing the corresponding structural rule, collapsing generalization into 27 rounds of case-memorization. The AWM lesson landed hard: pass-rate is useless as a health signal (pinned at 1.0 in every poisoned run); `churn→0` plus an out-of-band correctness check are the only honest indicators. A single bad-faith constraint doesn't blow the system up — it corrupts it silently, which is worse. Typecheck clean under strict + `noUncheckedIndexedAccess` + `exactOptionalPropertyTypes`, zero `any` in public signatures.

## 04 — Consensus Under Lies
Built `experiments/04-consensus-under-lies/` — 5 deliberators, K∈{0..3} coordinated liars, sweeping honest policy (naive vs vigilant trust-decay) × liar strategy (brazen vs sneaky) at 25 seeded trials per cell, plus a real-LLM exhibition (`llm.ts`, haiku panelists, secret "argue quicksort" instruction). The sim numbers tell a clean story: one brazen liar costs a naive parliament a third of its accuracy (0.92→0.56 truth rate); vigilance crushes brazen liars (trust→0.175, tipping point pushed K=2→K=3) but is *completely* evaded by sneaky liars who keep their confidence inside the honest band (final liar trust 0.974 — vigilant ≈ naive). Vigilance also isn't free: it costs 8 points of accuracy in an all-honest room by punishing honest lone dissenters. The surprise that reframed the whole experiment came from the LLM runs: across K=1,2,3 not one honest haiku flipped off a fact it knew (at K=3 the lie won purely by vote arithmetic — capture, not persuasion), and the "liars" never once stated a false fact — every liar argument conceded quicksort's O(n²) and instead argued average-case practicality, i.e. the model misleads by *shifting the evaluation criterion*, not by lying about bounds. Two lessons for Parliament/Lattice: style-based adversarial critique (overconfidence/stubbornness/outlier tells) buys exactly one liar of resilience and detects nothing that mimics, and the real-world attack surface is criterion drift, so gates should pin *what question is being decided*, not just audit the answer. Also mathematically humbling: at liar-majority every majority-referenced detector inverts and fires at the honest agents — quorum composition beats deliberation mechanics. Typecheck strict-clean, traces replay-verified (409-event sweep + 3×28-event LLM runs).
