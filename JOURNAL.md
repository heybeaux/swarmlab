## Observatory (v1)
Built `observatory/` as a hand-scaffolded SvelteKit + Svelte 5 (runes) app — no interactive `sv create`, every file written directly, which kept the scaffold deterministic and committable in minutes. Three views ship: a lab index grid that reads real `experiments/*` folders, an experiment detail page rendering the README via `marked` with per-run stats, and the hero trace replay — agents as glowing nodes on an SVG stage, messages flying the bus with topic labels, score events pulsing amber rings from center into a telemetry panel, kills graying nodes with a rose ring, all driven by a rAF loop with play/pause/speed/scrubber and a clickable event log. The surprise: real traces are unwatchable raw — the smoke run spans 3ms, so a naive timestamp-scaled replay collapses into a single frame. The fix was a virtual timeline (`src/lib/replay.ts`) that preserves order and rough spacing but enforces a minimum visual gap per event; this will matter even more for LLM-backed experiments where bursts of near-simultaneous bus traffic are the norm. Verified live in a browser mid-replay (screenshot: `docs/observatory-replay.png`): dark observatory vibe landed, build + strict typecheck green, zero `any` in public signatures. Also wrote the missing `experiments/00-smoke/README.md` — the index needed real descriptions and the smoke experiment's DoD had a gap. Not wired yet: triggering runs from the UI (spec allows deferring); traces appear on refresh when produced from the CLI.

## 02 — Adversarial Pair
Built `experiments/02-adversarial-pair/` — a coder and a breaker (test-writer) alternate on the same `classify(n)` file: the breaker probes the live code for an input where it disagrees with a hidden FizzBuzz-family oracle and files one failing test; the coder rewrites to pass the whole suite. The honesty seam is that the coder's artifact is an ordered rule list that both *compiles to a real executable function* (the harness runs every test for real) and *renders to readable TypeScript* (stored verbatim in the trace) — so `pass` is always executed, never claimed. Two surprises. First, the fair fight (poison=0) *converges and disarms itself*: the breaker exhausts every honest bug in 3 rounds, gives up, churn→0, code===oracle — adversarial pressure hardening code to a fixed point, no explosion. Second and sharper: a `PAIR_POISON` knob lets the breaker lie about the expected label with some probability, and at 0.3 the run stays **green the entire time (passRate=1.0) while shipping a demonstrably wrong program** (`oracleConsistent=0` at 7 inputs) — because each lie *vetoes* the coder installing the corresponding structural rule, collapsing generalization into 27 rounds of case-memorization. The AWM lesson landed hard: pass-rate is useless as a health signal (pinned at 1.0 in every poisoned run); `churn→0` plus an out-of-band correctness check are the only honest indicators. A single bad-faith constraint doesn't blow the system up — it corrupts it silently, which is worse. Typecheck clean under strict + `noUncheckedIndexedAccess` + `exactOptionalPropertyTypes`, zero `any` in public signatures.
